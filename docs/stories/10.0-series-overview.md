# Story 10.0: Epic 10 Overview - Developer Experience and Ergonomics

## Context / Background

fapilog is async-first and production-ready, but its developer experience is behind loguru and structlog. This epic focuses on reducing boilerplate and enabling progressive disclosure: simple defaults for new users, with advanced controls still available.

## Completed Features Summary

### 10.1 Configuration Presets
- Four built-in presets: `dev`, `production`, `fastapi`, `minimal`
- `preset` parameter in `get_logger()` and `get_async_logger()`
- Location: `src/fapilog/core/presets.py`

### 10.2 Pretty Console Output
- `StdoutPrettySink` for human-readable console output
- ANSI color codes for log levels with TTY auto-detection
- `format` parameter ("json", "pretty", "auto")
- Location: `src/fapilog/plugins/sinks/stdout_pretty.py`

### 10.3 One-Liner FastAPI Integration
- `setup_logging()` returns configured lifespan for FastAPI
- `get_request_logger` dependency for routes
- Automatic middleware registration and logger cleanup
- Location: `src/fapilog/fastapi/setup.py`, `src/fapilog/fastapi/dependencies.py`

## Scope (In / Out)
